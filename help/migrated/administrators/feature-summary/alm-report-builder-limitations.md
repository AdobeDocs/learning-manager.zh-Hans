# Adobe Learning Manager中的Report Builder限制

Report Builder涵盖了广泛的自定义报告需求，但此版本不支持某些报告类型、数据集和配置。 本节列出了已知的限制，并提供了可行的解决方法。

## 循环认证

在此版本中，某些涉及重复认证的查询无法在Report Builder中正常工作。 如果您的报告涉及循环认证周期，例如跨多个认证期间跟踪重新注册或重新完成，请改用Adobe Learning Manager中的现有认证报告。

## 不支持的数据集和报告类型

以下内容在此版本的Report Builder中不可用：

* **业务结果数据**：Report Builder不支持自带数据(BYOD)集成。 不支持将外部业务数据与LMS数据合并的报表。
* **审核报告**：无法在Report Builder中构建系统审核跟踪和管理员活动日志。 使用admin视图中的现有审核报告来获取此数据。
* **增量报告**：Report Builder每次都会生成完整快照报告。 不支持仅返回自上次下载以来更改的记录的增量报告。 对于增量数据，请使用作业API。

## 趋势报告限制

### 每个趋势报告仅有一个日期列

单个趋势报告只能按一个基于日期的列进行分组。 不能在同一报告中合并两个不同的基于日期的趋势。

例如，您可以构建：

* 显示&#x200B;**按月份**&#x200B;注册的报告（按注册日期分组）
* 显示按月&#x200B;**日的**&#x200B;完成情况（按完成日期分组）的单独报告

您无法构建将注册月份和完成月份都显示为单独趋势列的单一报告。 要获取两种报告，请创建两个单独的报告并排进行分析。

### 趋势反映的是当前数据，而不是历史快照

Report Builder中的趋势报告基于当前时间点数据，而不是历史快照。 该报告反映当今存在的记录在其日期字段中的状态。

这意味着：

* 于一月注册但随后取消注册的学习者可能不会显示在一月注册趋势
* 重新组织的用户组不会反映其过去几个月的历史会员资格
* 已删除的记录可能不会显示在其活动期间的趋势中

使用趋势报告进行方向分析。 为了精确历史记录或审计目的，请使用现有的固定报告或作业API导出。
